Project: SWARM (Sensory Web Analysis and Regression Mechanism)

Team Members:

  • Jonathan Ortiz Collazo (Lead)
  • Eric de Souza Piasentin

Description:

SWARM is a prototype project that aims to combine sensory data obtained from a swarm of IOT devices by utilizing client-server architecture and the cloud. These IOT devices collect sensory data like temperature, time, windspeed, humidity, and luminosity and transmit the data to a full-stack application which stores, analyzes and displays this data. The full stack application will be composed of a consumable API, a storage database, and publicly accessible website that provides visualizations.

Motivation:

Hurricane Dorian recently shocked the US with its potency and its destructive capabilities, leaving a strong impression on the team. We believe that collecting atmospheric data, performing data science techniques, and displaying actionable data are worthwhile endeavors. Sensory equipment can be acquired cheaply, and hosting software can be free if managed.

Proposal:

Jonathan, and Eric will divide their efforts among hardware and software requirements to complete this project. There is a safe target and stretch target of capabilities and implementation, both of which are attainable in a semester. In both cases the work will be divided such that Jonathan focuses on the web/server-related components, Eric will be focused on the hardware/sensor-related components. Each effort will be tracked, triaged, and tested throughout the semester.
